# Platinum (/p/OS)

64-bit OS written in C. Runs exclusively on QEMU x86_64. GPLv3.

## Stack

*   **Lang:** C99/C11 (Strict)
*   **Compiler:** GCC
*   **LibC:** musl
*   **License:** GPLv3
*   **Target:** QEMU x86_64 (`q35`, OVMF)
*   **Drivers:** VirtIO only (Net, Block, Console, RNG)
*   **AI Policy:** **HOSTILE.** Local OSS LLMs may be used for *syntax lookup* or *boilerplate scaffolding* only. **Zero tolerance** for committing unreviewed AI output. If your code smells like it was generated by a probabilistic token predictor without human understanding, it will be rejected. We build systems, we don't prompt them.

## Architecture

*   **Kernel:** Monolithic, static link. No modules.
*   **Init:** `pinit` (PID 1) -> `psup` (PID 2). Declarative services.
*   **FS:** `/p/fs` (custom log-structured CoW).
*   **Net:** Userspace TCP/IP (`netd`) over VirtIO-net.
*   **Userland:** GNU-like coreutils (`cat`, `ls`, `cp`, `mv`, `grep`, `sh`). Compatible flags where possible. Written by humans.

## Roadmap

### Phase 1: Kernel Base
- [ ] UEFI entry (x86_64 long mode)
- [ ] GDT, IDT, Paging (4KB pages)
- [ ] Memory Manager: Buddy allocator + Slab allocator
- [ ] VirtIO-console driver (early printk)
- [ ] Scheduler: Preemptive Round-Robin
- [ ] Syscalls: `read`, `write`, `open`, `close`, `fork`, `execve`, `exit`, `waitpid`
- [ ] ELF loader for static binaries

### Phase 2: Storage & Filesystem
- [ ] VirtIO-block driver
- [ ] VFS: Inodes, dentries, file descriptors
- [ ] `/p/fs`: Log-structured, copy-on-write, B-tree metadata
- [ ] `mkfs.pfs` utility
- [ ] Mount `/p/fs` as root

### Phase 3: Init & Shell
- [ ] `pinit`: Parse service structs, spawn processes
- [ ] `psup`: Supervise daemons, restart on crash
- [ ] `sh`: POSIX-compliant shell (pipes, redirects, variables, globbing)
- [ ] `login`: Basic auth against `/etc/passwd`
- [ ] `getty`: Spawn shell on VirtIO-console

### Phase 4: GNU-like Coreutils
- [ ] `cat`: Concatenate files
- [ ] `ls`: List directory contents (`-l`, `-a`, `-R`)
- [ ] `cp`: Copy files/dirs (`-r`, `-v`)
- [ ] `mv`: Move/rename files
- [ ] `rm`: Remove files/dirs (`-r`, `-f`)
- [ ] `mkdir`: Create directories (`-p`)
- [ ] `chmod`/`chown`: Permission management
- [ ] `grep`: Pattern matching (`-r`, `-i`, `-v`)
- [ ] `echo`, `pwd`, `cd`, `env`, `export`
- [ ] `sed`: Stream editor (basic substitution)
- [ ] `awk`: Pattern scanning (minimal implementation)

### Phase 5: Networking
- [ ] VirtIO-net driver
- [ ] `netd`: Userspace stack (ARP, IPv4, UDP, TCP)
- [ ] DHCP client in `netd`
- [ ] `ifconfig`/`ip`: Configure interfaces
- [ ] `ping`: ICMP echo request
- [ ] `sshd`: Port Dropbear or OpenSSH (linked against musl)
- [ ] `wget`/`curl`: Minimal HTTP client

### Phase 6: System Services & Polish
- [ ] `logd`: Syslog-compatible daemon
- [ ] `cron`: Task scheduler
- [ ] `ps`, `top`: Process monitoring
- [ ] `kill`, `nice`: Process control
- [ ] `man`: Manual page viewer
- [ ] Optimize kernel size (`-Os`, LTO)
- [ ] Final cleanup and documentation

## Build & Run

**Prereqs:** GCC, Python 3, QEMU, OVMF.

```bash
python3 build.py all
```

```bash
qemu-system-x86_64 \
    -machine q35 \
    -m 2G \
    -drive if=pflash,format=raw,readonly=on,file=/usr/share/OVMF/OVMF_CODE.fd \
    -drive if=virtio,format=raw,file=build/rootfs.img \
    -netdev user,id=n0 \
    -device virtio-net-pci,netdev=n0 \
    -device virtio-console-pci \
    -nographic \
    -kernel build/kernel.elf
```
